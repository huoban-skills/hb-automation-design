# 方案确认 / 分析结果 输出格式

"方案确认"（落地配置前）和"分析结果"（解读已有自动化）共用同一套主体结构，只是头部字段、标题和结尾话术按场景换。

## 通用主体（所有类型共用）

```text
<方案确认 | 分析结果>

<头部字段：按下方对应自动化类型填写>

执行流程：
主流程：
1. 节点：[中文节点名称]（[节点简述]）
   作用：[这个节点负责的动作]
   配置：[这个节点配置详情]
2. 节点：[中文节点名称]（[节点简述]）
   作用：[这个节点负责的动作]
   配置：[这个节点配置详情]

子流程（如涉及执行调用触发才写）：
调用的 call 自动化：xxx    # 分析阶段必填
传入参数：xxx              # 分析阶段必填
读取的输出变量：xxx        # 分析阶段必填
1. 节点：[中文节点名称]
   作用：[这个节点负责的动作]
   配置：[这个节点配置详情]

自检（方案阶段必带，对照 references/preflight-checklist.md 逐区给结论；未命中的区写"不涉及"）：
- 创建带明细的主单：[明细表 xxx 已作为子表预填（节点N）；子表数组来源：节点M「获取多条数据」 / 不涉及]
- 循环：[模式与嵌套结论 / 不涉及]
- 计算字段：[复用了哪个计算字段，或为何必须重算 / 不涉及]
- 第三方接口：[失败兜底与返回解析结论 / 不涉及]
- 结束与反馈：[1 个结束；失败路径如何反馈]
```

**两条硬规则：**

1. **节点名称用中文**，取自 `hac automation docs show node/index` 的"节点名称"列（如"获取单条数据""条件分支""数组循环""执行调用触发"），不得自行翻译或直接写 `type`。
2. **节点简述**不是节点的通用功能，而是用一句话概括它在**当前业务流程**里具体做的事（如"按订单号查询待发货明细""判断库存是否充足"），与 `作用` 互补——简述便于扫读，`作用` 展开细节。

**方案阶段**：结尾追加 `请确认以上方案，或告诉我需要调整的地方。`，等用户确认后再落地。
**分析阶段**：用真实字段名解释，不输出字段 ID；涉及执行调用触发时子流程块的调用元信息必填。

## 各类型头部字段

### 按钮（button）

```text
按钮名称：xxx
所在表格：xxx
触发方式：单条 / 批量
显示位置：详情页 / 列表 / 右键菜单
触发条件：xxx
关键判断字段：xxx
交互判断：是否需要打开页面给用户确认/补充字段？如果不用交互节点，原因是什么？（方案阶段必填，分析阶段省略）
```

### 调用触发（call）

```text
调用触发名称：xxx
所在表格：xxx
页面提示：xxx
提交按钮名称：xxx

触发参数：
- 参数名（类型，必填/选填）→ 来源说明

输出变量：
- 变量名（类型）→ 初始化方式，调用方如何读取
```

### 数据触发（item_trigger）

```text
数据触发名称：xxx
监控表格：xxx
触发动作：创建 / 修改 / 删除
指定触发字段：xxx（或不限）
权限组：xxx
触发条件：xxx
并发执行：是 / 否
```

### 自动填写（autofill）

```text
自动填写名称：xxx
监控字段：xxx
触发页面：创建页 / 编辑页 / 单字段编辑
触发条件：xxx
回填目标：xxx
```

自动填写主体后追加回填明细：

```text
回填明细：
- 字段名 ← 回填值来源
（若存在子表回填）子表回填：
- 目标子表：xxx
- 循环来源：xxx
- 每行映射：字段名 ← 值来源
（若触发源来自子表字段）触发字段归属：
- subtable_field_id → 主表子表字段名
```

### 定时触发（timer_trigger）

```text
定时触发名称：xxx
作用表格：xxx
开始时间：xxx
执行周期：xxx
触发条件：xxx
```
